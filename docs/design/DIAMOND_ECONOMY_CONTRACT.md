# 고양이 오디세이 — Diamond Economy Contract

## 무료 플레이 루프

모든 메인 스테이지와 엔드게임의 기본 입장은 무료다. 플레이 보상은 골드, 영웅 조각, 각성 재료, 유물, 보스 동료도와 외형이며 게임 안에서 다이아를 지급하지 않는다.

## 다이아가 필요한 순간

- 새로운 영웅과 편성 특성을 확보하고 싶을 때.
- 실패한 보스의 현재 상태에서 한 번 이어가고 싶을 때.
- 가호 선택이나 항로 상점을 다시 굴리고 싶을 때.
- 폭풍 항로 또는 토벌전의 추가 시도를 원할 때.
- 각성 재료가 조금 부족할 때.

## 고정 소비 액션

| actionId | 이름 | 비용 | 효과 | 반복 |
| --- | --- | ---: | --- | --- |
| `oracle-summon-1` | 신탁 소환 1회 | 100◆ | 영웅 또는 조각 1개 결과 | 예 |
| `oracle-summon-10` | 신탁 소환 10회 | 900◆ | 영웅/조각 10개, 천장 누적 | 예 |
| `battle-rescue` | 구조 요청 | 60◆ | 패배 직전 상태에서 파티 50% 회복 | 예 |
| `blessing-reroll` | 가호 재선택 | 30◆ | 현재 가호 후보 3개 재추첨 | 예 |
| `storm-extra-run` | 폭풍 항로 추가 출항 | 40◆ | 주간 무료 시도 이후 1회 추가 | 예 |
| `raid-extra-key` | 토벌 열쇠 보충 | 50◆ | 3파티 토벌 1회 추가 | 예 |
| `awakening-materials` | 각성 재료 보충 | 120◆ | 선택 영웅 각성 재료 묶음 | 예 |
| `vault-expansion` | 보물고 확장 | 180◆ | 유물 보관 슬롯 영구 +20 | 아니오 |

### 가호 재선택 규칙

- 폭풍 항로의 가호 선택 노드에서만 구매할 수 있다.
- 현재 세 후보와 겹치지 않는, 실제 전투 효과가 연결된 세 후보를 주간·회차·노드·재선택 횟수로 결정적으로 산출한다.
- 후보 계획은 save를 바꾸지 않으며, 동일한 `purchaseId`로 차감과 보상 커밋이 끝난 뒤에만 새 후보가 저장된다.
- 중단 복구와 연속 클릭은 같은 구매 의도를 재생하므로 한 번만 차감되고 한 번만 재선택된다.

### 보물고 규칙

- 신규 save의 기본 보관량은 20칸이며 기존 save에 기록된 더 큰 보관량은 줄이지 않는다.
- `vault-expansion`은 계정당 한 번만 구매할 수 있고 정확히 20칸을 영구 추가한다.
- 보물고가 찬 상태에서 새 유물을 얻으면 유물 등급에 따른 유물 가루로 자동 전환되며, 보상 영수증에 `vault_full` 사유를 남긴다.
- 영구 상품 영수증은 반복 구매 영수증의 순환 보관 한도를 넘어도 삭제하지 않는다.

## 확률과 천장

- 1회 결과 확률과 배너 풀을 소환 확인 화면에서 표시한다.
- 35회부터 최고 희귀도 확률이 상승하고 45회에 최고 희귀도 1개를 보장한다.
- 중복은 영웅 조각과 등급별 운명 가루로 전환된다. 운명 가루 100개는 다음 ★5를 픽업 영웅으로 확정하는 데 사용한다.
- 구매 버튼은 pending 상태에서 잠그고, 같은 의도에는 같은 idempotency key를 사용한다.
- 차감 성공 후 보상 지급과 저장이 실패하면 pending 구매를 다음 시작에 복구한다.

## Host 경계

- 외부 호출은 `wallet.getBalance`, `wallet.spend`만 사용한다.
- `id`, `amount`, `reason`, `requiresConfirm`은 manifest와 정확히 일치한다.
- 지갑 잔액을 게임 save에 기록하지 않는다.
- 잔액 부족 UI에는 보유량·필요량·확인만 표시한다.
